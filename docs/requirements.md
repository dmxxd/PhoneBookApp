# Функциональные требования (ФТ)

### Для пользователя

* **ФТ-01:** Регистрация. Пользователь может создать новую учетную запись, указав имя пользователя, email и пароль.
* **ФТ-02:** Авторизация. Пользователь может войти в систему, используя зарегистрированный email и пароль.
* **ФТ-03:** Отображение списка своих контактов. После входа в систему пользователю отображается список всех его сохраненных контактов.
* **ФТ-04:** Добавление нового контакта. Пользователь может создать новый контакт, заполнив его данные (обязательное поле — имя).
* **ФТ-05:** Редактирование своего контакта. Пользователь может открыть существующий контакт и изменить его поля (телефон, email, адрес, заметку и т.д.).
* **ФТ-06:** Удаление своего контакта с подтверждением. При попытке удалить контакт система запрашивает подтверждение действия, чтобы избежать случайного удаления.
* **ФТ-07:** Поиск контакта по имени, фамилии или телефону среди своих. Пользователь может ввести поисковый запрос в главном окне, чтобы отфильтровать свои контакты по совпадению в указанных полях.
* **ФТ-08:** Просмотр всех полей контакта. Пользователь может открыть контакт и просмотреть всю сохраненную информацию о нем в детальном виде.
* **ФТ-09:** Добавление тегов к контакту при создании/редактировании. При создании или редактировании контакта пользователь может выбрать один или несколько тегов для его категоризации (например: Работа, Семья, Друзья).
* **ФТ-10:** Фильтрация списка контактов по одному или нескольким тегам. Пользователь может выбрать конкретный тег (или несколько тегов), чтобы отфильтровать список и отобразить только контакты с этими тегами.
* **ФТ-11:** Указание даты рождения контакта. В форме редактирования/создания контакта присутствует календарь, позволяющий сохранить дату рождения контакта.
* **ФТ-12:** Получение всплывающих сообщений о днях рождения в ближайшие 3 дня при входе в систему. При успешной авторизации система автоматически проверяет и выводит уведомление о предстоящих днях рождения контактов в ближайшие 3 дня.
* **ФТ-13:** Переключение между светлой и тёмной темой оформления. Пользователь может переключить визуальную тему интерфейса приложения (светлая/темная) по своему желанию.

### Для администратора

* **ФТ-14:** Отображение списка всех зарегистрированных пользователей. Администратор может открыть окно управления и увидеть полный список всех пользователей, зарегистрированных в системе.
* **ФТ-15:** Добавление нового пользователя. Администратор может создавать новых пользователей напрямую, минуя стандартную форму регистрации.
* **ФТ-16:** Блокировка любого пользователя. Администратор может заблокировать выбранного пользователя. Заблокированный пользователь теряет возможность входить в систему.
* **ФТ-17:** Разблокировка любого пользователя. Администратор может снять блокировку с ранее заблокированного пользователя, восстановив ему доступ к системе.
* **ФТ-18:** Изменение роли пользователя. Администратор может назначить любому пользователю роль «Администратор» или вернуть роль «Обычный пользователь».
* **ФТ-19:** Удаление пользователя. Администратор может полностью удалить учетную запись пользователя и все связанные с ним данные из базы.
* **ФТ-20:** Просмотр контактов любого пользователя. Администратор имеет привилегированный доступ к просмотру контактной книги любого пользователя системы.
