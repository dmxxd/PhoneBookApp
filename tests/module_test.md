# Модульное тестирование

## 1. Тест-кейсы модульного тестирования окна авторизации

| ID | Входные данные | Последовательность действий | Результаты выполнения |
| :---: | :--- | :--- | :--- |
| 1 | Корректный email и пароль | 1) Ввести email<br>2) Ввести пароль<br>3) Нажать «Войти» | Выполняется авторизация, открывается главное окно |
| 2 | Неверный пароль | 1) Ввести email<br>2) Ввести неверный пароль<br>3) Нажать «Войти» | Отображается сообщение об ошибке авторизации |
| 3 | Пустой email | 1) Оставить поле email пустым<br>2) Нажать «Войти» | Отображается предупреждение о заполнении полей |
| 4 | Пустой пароль | 1) Оставить поле пароля пустым<br>2) Нажать «Войти» | Отображается предупреждение о заполнении полей |
| 5 | Переход к регистрации | 1) Нажать кнопку «Регистрация» | Открывается окно регистрации |

## 2. Тест-кейсы модульного тестирования окна регистрации

| ID | Входные данные | Последовательность действий | Результаты выполнения |
| :---: | :--- | :--- | :--- |
| 1 | Корректные данные | 1) Заполнить все поля<br>2) Нажать «Регистрация» | Пользователь успешно зарегистрирован |
| 2 | Пароль менее 4 символов | 1) Ввести короткий пароль<br>2) Нажать «Регистрация» | Отображается сообщение о минимальной длине пароля |
| 3 | Разные пароли | 1) Ввести разные значения в поля пароля<br>2) Нажать «Регистрация» | Отображается сообщение «Пароли не совпадают» |
| 4 | Некорректный email | 1) Ввести некорректный email<br>2) Нажать «Регистрация» | Отображается сообщение об ошибке email |
| 5 | Отмена регистрации | 1) Нажать «Отмена» | Окно регистрации закрывается |

## 3. Тест-кейсы модульного тестирования главного окна

| ID | Входные данные | Последовательность действий | Результаты выполнения |
| :---: | :--- | :--- | :--- |
| 1 | Существующие контакты | 1) Выполнить вход в систему | Список контактов загружается и отображается |
| 2 | Новый контакт | 1) Нажать «Добавить контакт» | Открывается окно добавления / редактирования контактов |
| 3 | Поисковый запрос | 1) Ввести текст в поле поиска | Выполняется фильтрация контактов |
| 4 | Учётная запись администратора | 1) Войти под администратором<br>2) Нажать «Управление пользователями» | Открывается окно управления пользователями |
| 5 | Выход из системы | 1) Нажать пункт меню «Выход» | Открывается окно авторизации, главное окно закрывается |

## 4. Тест-кейсы модульного тестирования окна добавления контакта

| ID | Входные данные | Последовательность действий | Результаты выполнения |
| :---: | :--- | :--- | :--- |
| 1 | Корректные данные контакта | 1) Заполнить обязательные поля<br>2) Нажать «Сохранить» | Контакт успешно сохраняется |
| 2 | Пустое имя контакта | 1) Оставить поле имени пустым<br>2) Нажать «Сохранить» | Отображается сообщение «Введите имя» |
| 3 | Редактирование контакта | 1) Открыть существующий контакт | Поля формы заполняются данными контакта |
| 4 | Выбор тегов | 1) Отметить теги<br>2) Нажать «Сохранить» | Теги сохраняются для контакта |
| 5 | Отмена ввода | 1) Нажать «Отмена» | Форма закрывается без сохранения |

## 5. Тест-кейсы модульного тестирования окна управления пользователями

| ID | Входные данные | Последовательность действий | Результаты выполнения |
| :---: | :--- | :--- | :--- |
| 1 | Список пользователей | 1) Открыть форму управления пользователями | Отображается список пользователей |
| 2 | Выбранный пользователь | 1) Выбрать пользователя<br>2) Нажать «Заблокировать» | Пользователь блокируется |
| 3 | Заблокированный пользователь | 1) Выбрать пользователя<br>2) Нажать «Разблокировать» | Пользователь разблокируется |
| 4 | Пользователь | 1) Выбрать пользователя<br>2) Нажать «Сделать администратором» | Пользователю назначается роль администратора |
| 5 | Пользователь | 1) Выбрать пользователя<br>2) Нажать «Удалить» | Пользователь удаляется из системы |
